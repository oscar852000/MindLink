# MindLink - 产品规格文档

> 一个只会聆听、整理、表达的超能力助手——不加见解，只让你的想法保持清醒。

---

## 1. 产品定位

### 一句话描述
将人类零散的想法持续整理成清醒的认知，并按需输出不同风格的表达。

### 核心价值
| 人类短板 | AI 能力 |
|----------|---------|
| 记忆有限、容易丢失 | 无限记忆 |
| 整理费力、逻辑混乱 | 自动归纳 |
| 表达消耗、风格单一 | 弹性输出 |

### 产品隐喻
> 超能力的人：只聆听和提问，不加见解，拥有无穷的记忆力、整理能力、表达能力。

---

## 2. 核心能力（两大模块）

### 2.1 整理（Crystallize）

**用户行为**：不断投喂零散想法

**AI 行为**：
- 抓核心、去噪、去重、更新
- 以当前时间为覆盖权重
- 保留"最清晰、准确、简洁的归档"
- 维护时间线与可追溯演变

**整理原则**：

| 处理方式 | 内容类型 |
|----------|----------|
| 忽略 | 牢骚、啰嗦、重复、跑题 |
| 记录 | 观点、最新认知、亮点创意 |
| 强调 | 强化核心的关键细节 |

### 2.2 输出（Express）

**用户行为**：请求表达想法

**AI 行为**：
- 基于同一个想法，按用户目的生成不同表达
- 输出可弹性控制：超短 / 标准 / 详细

**输出场景示例**：

| 场景 | 风格 | 长度 |
|------|------|------|
| 给投资人 | 商业、愿景导向 | 精炼 |
| 给程序员 | 技术、结构化 | 详细 |
| 给朋友 | 口语、轻松 | 简短 |
| 电梯稿 | 精准、有力 | 两三句 |
| 完整说明 | 全面、细节 | 长文 |

## 3. 产品原则（系统铁律）

1. **AI 是编辑器，不是作者**
   - 不妄做事实、不加个人观点
   - 只基于用户投喂与确认内容

2. **一个 Mind = 一个核心话题**
   - 对话补充时必须围绕该话题
   - 保持唯一性与清醒度

3. **时间覆盖权重**
   - 新认知更重要
   - 但历史可追溯、可回滚

4. **确认机制慎用**
   - 仅在冲突/删除/高风险改写时触发确认
   - 避免破坏自动化手感

## 4. 核心对象（数据模型）

| 对象 | 定义 |
|------|------|
| **Mind** | 一个想法档案（标题 + 可选核心目标） |
| **FeedItem** | 用户投喂的原始片段（只记录） |
| **Crystal** | 该 Mind 的"结构化清醒档案" |
| **Timeline** | 按时间记录 feed/整理/确认/输出等事件 |
| **OutputTask** | 一次输出请求（目的 + 风格 + 长度） |

**后期预留**：MindLink（想法与想法之间的逻辑关联）

---

## 5. Crystal 结构（锚点）

Crystal 以结构化内容维护，可渲染成 Markdown 展示：

```markdown
## 核心目标
[一句话，保持焦点]

## 当前认知
- 要点1（事实/观点/决策/假设）
- 要点2
- ...

## 亮点创意
[可强化核心或创新点的细节池]

## 待确认/待解决
[必要时由 AI 提问澄清]

## 演变记录
[每次关键变化摘要，版本化]
```

## 6. 整理器（Organizer）规则

对每次新投喂，整理器生成以下四类动作：

| 动作 | 说明 | 是否需确认 |
|------|------|-----------|
| **Add** | 新增观点/信息/细节 | 否 |
| **Refine** | 同一观点更清晰（不改变原意） | 否 |
| **Conflict** | 与旧认知互斥 | 是（以哪个为准？） |
| **Obsolete** | 过期或被新决策替代 | 是（确认归档） |

**噪声处理**：
- 牢骚、啰嗦、纯重复、跑题：保留在 Feed/Timeline，不进入 Crystal
- 可能强化核心/亮点的细节：进入"亮点创意"池，不干扰主线清醒度

## 7. 用户流程

### 7.1 创建 Mind
用户创建一个想法档案（标题 + 可选核心目标）

### 7.2 投喂（默认静默）
- 用户输入片段
- 系统立即返回极简确认：`✓ 已记录`
- 异步触发整理

### 7.3 查看总览
- 展示当前 Crystal（最清醒版本）
- 下方显示轻量提示："最近更新：整合了 3 条投喂"

### 7.4 对话补充（用户主动开启）
- AI 仅围绕该 Mind 的待确认问题进行提问
- 会话结束后生成"补充摘要"，更新 Crystal

### 7.5 输出提取
用户指定"目的/风格/长度"，系统基于 Crystal 生成表达

## 8. 产品结构

```
MindLink
├── 首页
│   ├── 创建 Mind（新想法）
│   └── 想法目录（已有想法列表）
│
└── Mind 详情页
    ├── 投喂区（输入新想法片段）
    ├── 总览区（Crystal，最清醒的想法全貌）
    ├── 输出区（按要求生成表达）
    └── [后续] 时间轴 / 对话补充
```

## 9. AI 行为规则

### 整理时

```yaml
角色: 只聆听，不评判
目标: 保持想法清醒、完整、准确

做:
  - 提取观点和认知
  - 新内容覆盖旧内容（时间权重）
  - 保留强化核心的细节
  - 保留亮点创意

不做:
  - 不加入自己的见解
  - 不保留牢骚和情绪
  - 不保留重复表述
  - 不主动询问（除非严重模糊或冲突）
```

### 输出时

```yaml
角色: 表达助手
目标: 按用户要求转化想法

可调整:
  - 受众: 投资人/技术/朋友/自己
  - 长度: 一句话/一段/完整
  - 风格: 正式/轻松/结构化

保持:
  - 忠于想法内容
  - 不添加未投喂的信息
  - 不改变用户的立场
```

- 不在输出中暴露
