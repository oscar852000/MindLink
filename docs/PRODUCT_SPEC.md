# MindLink - 产品说明文档

> 一个只会聆听、整理、表达的超能力助手——不加见解，只让你的想法保持清醒。

---

## 1. 产品定位

### 一句话描述
将人类零散的想法持续整理成清醒的认知，并按需输出不同风格的表达。

### 核心价值
| 人类短板 | AI能力 |
|----------|--------|
| 记忆有限、容易丢失 | 无限记忆 |
| 整理费力、逻辑混乱 | 自动归纳 |
| 表达消耗、风格单一 | 弹性输出 |

### 产品隐喻
> 超能力的人：只聆听和提问，不加见解，拥有无穷的记忆力、整理能力、表达能力。

---

## 2. 核心功能

### 2.1 整理（核心）

**用户行为**：不断投喂零散想法

**AI行为**：
- 以当前时间为覆盖权重
- 归纳、补充、更新、迭代
- 始终保持一份"最清醒"的想法全貌

**整理原则**：

| 处理方式 | 内容类型 |
|----------|----------|
| 忽略 | 牢骚、啰嗦、重复表述 |
| 记录 | 观点、最新认知、亮点创意 |
| 强调 | 强化核心的关键细节 |

**时间权重规则**：
- 新认知默认覆盖旧认知
- 除非用户明确表示"这是补充而非替代"

---

### 2.2 输出（核心）

**用户行为**：请求表达想法

**AI行为**：
- 根据目标受众调整风格
- 根据要求调整长度
- 根据场景调整正式程度

**输出场景示例**：

| 场景 | 风格 | 长度 |
|------|------|------|
| 给程序员 | 技术、结构化 | 详细 |
| 给投资人 | 商业、愿景导向 | 精炼 |
| 给朋友 | 口语、轻松 | 简短 |
| 自我回顾 | 完整、细节 | 完整 |

---

### 2.3 对话补充（辅助）

**触发条件**：用户主动开启对话模式

**AI行为**：
- 围绕当前想法提问澄清
- 始终不偏离核心话题
- 补充信息后整合进想法

**设计原则**：
- 慎用确认机制
- 避免每一步都问，破坏自动化感觉
- 只在真正模糊时触发

---

## 3. 产品结构

### 信息架构

```
MindLink
├── 首页
│   ├── 创建 Mind（新想法）
│   └── 想法目录（已有想法列表）
│
└── Mind 详情页
    ├── 投喂区（输入新想法片段）
    ├── 总览区（当前最清醒的想法全貌）
    ├── 输出区（按要求生成表达）
    └── [后续] 时间轴 / 对话补充
```

### 核心概念

| 概念 | 定义 |
|------|------|
| Mind | 一个封闭的想法空间（一个话题一个Mind） |
| 投喂 | 用户输入的原始想法片段 |
| 总览 | AI整理后的"最清醒"版本 |
| 输出 | 按用户要求生成的表达 |

---

## 4. MVP 范围

### 只做三件事

1. **投喂** → 输入想法（文字）
2. **总览** → 看当前"最清醒"的想法全貌
3. **输出** → 按要求生成表达

### MVP 用户流程

```
[创建 Mind] "AI想法助手"
     ↓
[投喂] "我想做一个记录想法的工具..."
     ↓
[投喂] "应该有整理和输出两个核心功能..."
     ↓
[投喂] "输出可以有不同风格"
     ↓
[投喂] "风格功能先不做了，太复杂"
     ↓
[看总览]
┌─────────────────────────────────────┐
│ 核心：AI想法助手                      │
│ 目标：记录 + 整理 + 输出              │
│ 当前决策：风格功能暂缓                │
│ 关键细节：...                        │
│                                     │
│ ⓘ 最近更新：整合了4条投喂            │
└─────────────────────────────────────┘
     ↓
[输出] "写一段给程序员看的需求说明"
     ↓
[AI生成] → 技术风格的项目说明
```

### 暂不做的功能

| 功能 | 原因 | 计划 |
|------|------|------|
| 每次整理确认 | 破坏自动化体验 | 暂不做 |
| 详细时间轴 | 非核心 | 第二期 |
| 对话补充 | 非核心 | 第二期 |
| 跨Mind链接 | 复杂 | 第三期 |
| 语音投喂 | 需额外开发 | 第三期 |

## 5. AI 行为规则

### 整理时

```yaml
角色: 只聆听，不评判
目标: 保持想法清醒、完整、准确

做:
  - 提取观点和认知
  - 新内容覆盖旧内容（时间权重）
  - 保留强化核心的细节
  - 保留亮点创意

不做:
  - 不加入自己的见解
  - 不保留牢骚和情绪
  - 不保留重复表述
  - 不主动询问（除非严重模糊）
```

### 输出时

```yaml
角色: 表达助手
目标: 按用户要求转化想法

可调整:
  - 受众: 技术/商业/朋友/自己
  - 长度: 一句话/一段/完整
  - 风格: 正式/轻松/结构化

保持:
  - 忠于想法内容
  - 不添加未投喂的信息
  - 不改变用户的立场
```

---

## 6. 整理透明度

采用**轻量展示**模式：

- 总览下方显示一行提示
- 例如："最近更新：整合了3条投喂，移除了重复表述"
- 给用户"AI在工作"的感知，但不打扰

---

## 7. 后续迭代方向

### 第二期
- 时间轴展示（查看想法演变）
- 对话补充模式（AI提问澄清）
- 历史版本对比

### 第三期
- 跨Mind链接（想法间的关联）
- 语音投喂
- 导出功能（Markdown/PDF）

### 远期
- 多Mind智能关联
- 自动发现想法间的矛盾
- 定期"反刍提醒"

---

## 8. 敏感信息处理

如用户投喂账号密码等敏感信息：
- 提示"建议使用专业密码管理工具"
- 如用户坚持，标记为敏感内容
- 不在输出中暴露
